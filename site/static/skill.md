# Arango documentation skill

Use this skill when working with ArangoDB, the Arango Contextual Data Platform,
the Arango Managed Platform (AMP), the Platform Suite, or the Agentic AI Suite.

## Discovery
- Documentation index: https://docs.arango.ai/llms.txt
- Full-text corpus: https://docs.arango.ai/llms-full.txt
- Markdown of any page: append `.md` to its URL.

## Where to look
- Contextual Data Platform (entity-aware retrieval, graph reasoning, temporal state, governance): https://docs.arango.ai/contextual-data-platform/
- Agentic AI Suite (Ada, AutoGraph, GraphRAG, GraphML): https://docs.arango.ai/agentic-ai-suite/
- Platform Suite (Kubernetes operations, Graph Visualizer, Query Editor): https://docs.arango.ai/platform-suite/
- ArangoDB database (AQL, deployment, HTTP API): https://docs.arango.ai/arangodb/stable/
- Arango Managed Platform (AMP): https://docs.arango.ai/amp/
- Ecosystem (drivers, integrations): https://docs.arango.ai/ecosystem/
- ArangoDB MCP Server (natural-language AQL against your own database): https://docs.arango.ai/ecosystem/arangodb-mcp-server/

## Conventions
- For the ArangoDB database, prefer the `stable` version path; use `devel` for the
  in-development release. Platform and Agentic AI Suite docs track v4.0.
- AQL is ArangoDB's query language; the MCP server can generate and execute AQL
  from natural language and ground it in your database schema.
